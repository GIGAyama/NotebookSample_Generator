# Google ドライブ同期の設定手順

このツールは、保存したノート（板書計画）を **Google ドライブへ自動バックアップ**し、
別の端末でも同じ Google アカウントでログインするだけで取り込めます。

- サーバー（バックエンド）は不要です。ブラウザから直接 Google の API を呼び出します。
- アクセス権限は `drive.file` スコープ（**このアプリが作成したファイルだけ**）に限定しています。
  ユーザーの他のドライブのファイルには一切アクセスしません。
- バックアップは `notebook_sample_backup.json` というファイル名で保存されます。

この機能を有効にするには、Google Cloud Console で **OAuth クライアントID** を 1 つ作成し、
アプリのビルド時に `VITE_GOOGLE_CLIENT_ID` として渡すだけです。
（Client ID は公開情報でありシークレットではありません。ブラウザに埋め込まれても問題ありません。）

未設定のままでもアプリは通常どおり動作し、「データ管理」画面の同期欄が
「初期設定が必要です」という表示になるだけです。

---

## 1. Google Cloud プロジェクトを用意する

1. [Google Cloud Console](https://console.cloud.google.com/) にログイン。
2. 画面上部からプロジェクトを新規作成（既存のものでも可）。

## 2. Google Drive API を有効化する

1. 「API とサービス」→「ライブラリ」を開く。
2. **Google Drive API** を検索して「有効にする」。

## 3. OAuth 同意画面を設定する

1. 「API とサービス」→「OAuth 同意画面」を開く。
2. User Type は **外部** を選択。
3. アプリ名（例: `ノート見本作成ツール`）、サポートメール、デベロッパー連絡先を入力。
4. スコープの追加で以下を追加（`drive.file` は「機密性の高いスコープ」に分類されます）:
   - `.../auth/userinfo.email`
   - `openid`
   - `.../auth/drive.file`
5. まず自分のアカウントで試す場合は、公開ステータスを **テスト** のままにして
   「テストユーザー」に自分（および同僚）の Google アカウントを追加します。
   - 広く一般に公開して使ってもらう場合は、後日 **本番環境に公開**し、
     Google の審査（`drive.file` は機密スコープのため確認が必要な場合あり）を通してください。
     審査前でも「テスト」中はテストユーザーなら利用できます。

## 4. OAuth クライアントID（ウェブアプリケーション）を作成する

1. 「API とサービス」→「認証情報」→「認証情報を作成」→「OAuth クライアント ID」。
2. アプリケーションの種類: **ウェブ アプリケーション**。
3. **承認済みの JavaScript 生成元** に、アプリを配信する URL の「オリジン」を追加します
   （パスは付けません）:
   - 本番（GitHub Pages）: `https://gigayama.github.io`
   - ローカル開発: `http://localhost:5173`
   > このツールはポップアップ（トークン）方式のため、**リダイレクト URI の登録は不要**です。
   > JavaScript 生成元だけで動作します。
4. 作成後に表示される **クライアント ID**（`xxxxxxxx.apps.googleusercontent.com`）を控えます。

## 5. クライアントID をアプリに渡す

### 本番（GitHub Pages / GitHub Actions）

1. GitHub リポジトリの **Settings → Secrets and variables → Actions → Variables** タブを開く。
2. 「New repository variable」で以下を追加:
   - Name: `VITE_GOOGLE_CLIENT_ID`
   - Value: 手順 4 で控えたクライアント ID
3. `main` ブランチへの push（またはワークフローの手動実行）で再デプロイすると有効になります。

> Client ID は公開情報なので Secret ではなく **Variable** で問題ありません。

### ローカル開発

```bash
cp .env.example .env
# .env を編集して VITE_GOOGLE_CLIENT_ID にクライアントIDを設定
npm install
npm run dev
```

---

## 使い方（利用者向け）

1. 画面右上の「データ管理」（`Alt+3`）を開く。
2. 「Google ドライブ同期」の **「Google でログインして同期」** を押してログイン・許可。
3. 以降、ノートを保存すると自動でバックアップされます。
4. 別の端末では、同じアカウントでログインするだけで既存のノートが取り込まれます。
   手動で取り込みたいときは「今すぐ同期」を押してください。

### 補足・仕様

- 同期対象は「保存されたノート一覧」です（編集中の作業内容そのものは同期しません）。
- 同じ内容を両端末で編集した場合は、**より新しい保存日時のノートが優先**されます。
- 取りこぼしを防ぐため、どちらか一方にしか無いノートは必ず残す統合方式です。
  そのため **削除は自動同期されません**（片方で消しても、もう片方に残っていれば復活します）。
  完全に消したい場合は各端末で削除してください。
